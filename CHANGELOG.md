# Changelog

## 2026-07-05

- **Fix**: `leads` и `daily_stats` синки падали со 100% ошибок с 2026-07-01 (429 от PlusVibe) —
  причина: рост числа лидов с ~14k до 36k+ дольше держал пагинацию в зоне риска, плюс в
  `daily_stats` вообще не было паузы между запросами, плюс общий клиент не делал retry на 429
  (первая же ошибка валила весь прогон целиком). Исправлено:
  - `lib/plusvibe-api.js`: retry-backoff на 429 (5 попыток, 2s→32s экспоненциально)
  - `getLeads`: лимит страницы 100 → 1000 (максимум по API), ~10x меньше запросов
  - `daily_stats.js`: добавлена пропущенная задержка 210ms между кампаниями
- **Removed**: `campaign_stats`, `warmup_stats`, `lead_status_counts` — ни одна таблица нигде не
  читалась (проверено grep по всему `outreach-cockpit`), при этом `campaign_stats` каждые 30 мин
  тянул 6-месячный диапазон по каждой кампании ради одной строки за сегодня.
- **Restored**: `syncTags` — молча выпал из раннера с 2026-06-07 (никто не заметил), возвращён в
  `all`/добавлена отдельная команда `tags`. Дешёвый (1 запрос), держит актуальными имена тегов для
  `account_tags`.
- **Removed**: черновик `fathom/sync.js` (ветка от 2026-06-22) — никогда не коммитился и не
  деплоился, дублировал уже рабочий синк в `outreach-cockpit/server/modules/fathom/sync.ts`.
- Смёржен зависший локальный VPS-коммит `06d1440` (2026-07-03, никогда не пушился на GitHub):
  добавил `node index.js health` (проверка staleness по `sync_log`) и фикс `emails_sync`,
  который выпадал из cron с ~апреля (600ms delay + retry-backoff вместо 210ms без ретраев).
- Added `README.md` — таблица всех синков (частота/механика/потребители/статус).
